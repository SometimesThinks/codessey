### Docker 바인드 마운트 실행 및 전후 비교

`Dockerfile` 작성 후 `app/index.html` 코드를 변경했으나 결과에 반영되지 않는 문제 발생.

- **원인:** `COPY` 명령어는 이미지 빌드 시점에서 파일을 복사하므로, 빌드 이후 로컬에서 수정한 코드는 이미 실행 중인 컨테이너나 기존 이미지에 반영되지 않음
- **해결:** 실시간 코드 반영이 필요한 개발 환경에서는 **Bind Mount**(`-v` 옵션)를 사용하여 로컬 디렉토리를 컨테이너 내부에 직접 마운트

```zsh
$ docker run -d -p 8000:80 -v $(pwd)/app:/usr/share/nginx/html --name bind_test my_image

# 컨테이너 상태 확인
$ docker ps
CONTAINER ID   IMAGE      COMMAND                  CREATED          STATUS          PORTS                                   NAMES
bae1fecf71dd   my_image   "/docker-entrypoint.…"   5 seconds ago    Up 4 seconds    0.0.0.0:8000->80/tcp, [::]:8000->80/tcp   bind_test

# 최초 curl 요청 확인
$ curl http://localhost:8000/
<!doctype html>

    <style>

        .card { text-align: center; padding: 2.5rem; border-radius: 1rem; background: white; box-shadow: 0 4px 10px rgba(0,0,0,0.05); border: 1px solid #e2e8f0; }
        h1 { margin: 0; font-size: 2.5rem; color: #6366f1; }

    </style>

    <div class="card">

        <p>최소한의 디자인으로 구성된 정적 페이지입니다.</p>

</html>

# index.html 수정 후 다시 curl 요청 (빌드 없이 즉시 반영 확인)
$ curl http://localhost:8000/

```

### Docker 볼륨 영속성 검증

```zsh
# 볼륨 생성
$ docker volume create test_volume
test_volume

# 볼륨을 마운트하여 컨테이너 실행 및 파일 생성
$ docker run -it -v test_volume:/test_volume --name volume_test ubuntu
root@6c08a47dba29:/# ls
bin boot dev etc home lib lib64 media mnt opt proc root run sbin srv sys test_volume tmp usr var
root@6c08a47dba29:/# echo "Is this file alive?" > /test_volume/test.txt
root@6c08a47dba29:/# cat /test_volume/test.txt
Is this file alive?
root@6c08a47dba29:/# exit

# 기존 컨테이너 삭제
$ docker rm volume_test
volume_test

# 생성했던 파일이 남아있는지 새 컨테이너에서 확인
$ docker run -it -v test_volume:/test_volume --name volume_check ubuntu
root@2781d87df423:/# cat /test_volume/test.txt
Is this file alive?
```

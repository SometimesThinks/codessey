# Mini NPU Simulator 개발

## 프로젝트 개요

간단한 NPU의 핵심 연산인 MAC 연산을 시뮬레이션하고, 입력된 패턴이 어떤 필터와 더 유사한지 판정하며 성능을 분석하는 프로그램입니다.

## 실행 환경

- Python 3.12.1

## 프로젝트 구조

```text
.
├── main.py          # 프로그램 진입점
├── app.py           # 모드 분기 및 전체 실행 흐름
├── cli.py           # 터미널 입력/출력 함수
├── dataset.py       # data.json 로드, 전처리, 라벨 정규화
├── simulator.py     # MAC 연산, 판정, 성능 측정 로직
├── utils.py         # 입력 보조 유틸리티
├── constants.py     # EPSILON, 반복 횟수 등 전역 상수 관리
├── data.json        # 모드 2 전용 분석 데이터셋 (15개 케이스)
├── input.txt        # 개발용 자동 입력 더미 파일
└── README.md        # 프로젝트 문서
```

## 실행 방법

```text
프로젝트 루트에서 `python3 main.py` 명령어를 실행하여 프로그램을 시작합니다.
```

## 요구 사항 및 기능 목록

### 데이터 구조

- [x] NxN 2차원 패턴 및 필터 저장
- [x] 특정 위치의 값 저장 및 읽기
- [x] 3x3, 5x5, 13x13, 25x25 크기 처리

### 모드 1 입력 처리(3x3)

- [x] 3x3 필터 한 줄씩(공백 구분) 입력 및 저장
- [x] 3x3 패턴 한 줄씩(공백 구분) 입력 및 저장
- [x] 행/열 수 불일치, 숫자 파싱 실패 시 안내 문구 출력 및 재입력 유도

### 모드 2 JSON 로드 및 스키마 검증(data.json)

- [x] data.json 파일 로드 및 전처리 (수치 데이터 float 일괄 변환)
- [x] filters: size_3, size_5, size_13, size_25 지원
- [x] patterns: size*{N}*{idx} 키 구조 지원 (정교한 2단계 숫자 정렬 적용)
- [x] 필터와 패턴 크기 일치 검증 및 예외 처리

### 라벨 정규화(표준화)

- [x] Cross, X 표준 라벨 사용
- [x] '+', 'cross' -> Cross / 'x' -> X 정규화
- [x] 'undecided' -> UNDECIDED 표준 상태 정규화

### MAC 연산

- [x] 입력 패턴과 필터 위치별 곱셈 후 덧셈 (O(N^2))
- [x] NumPy 등 외부 라이브러리 사용 금지 및 순수 파이썬 구현

### 판정 및 출력 규칙

- [x] 허용 오차(epsilon = 1e-9) 기반 동점 판정 정책
- [x] 모드 1 판정: `A`, `B`, `UNDECIDED`
- [x] 모드 2 출력 구조: `판정(X/Cross/-)`, `기대값`, `결과(PASS/FAIL)`
- [x] 기대값이 'UNDECIDED'인 경우, 동점 발생 시 PASS 처리

### 성능 분석 및 결과 리포트

- [x] 크기별 MAC 연산 100회 반복 평균 시간 측정 (ms 단위)
- [x] 규격별 성능 통계 표(Table) 출력
- [x] 모드 2 전체 테스트 통계 및 실패 케이스 사유 요약 브리핑

## 결과 리포트 및 분석

### 1. 실패 사유 세부 분석 (Failure Analysis)

시뮬레이션 수행 결과 발생하는 `FAIL` 케이스는 다음 4가지 핵심 사유로 분류됩니다:

- **동점 발생 (Undecided Case)**: `Cross`와 `X` 필터의 점수 차이가 `1e-9` 이하일 때 발생하며, 정답 데이터의 기대값이 `UNDECIDED`가 아닌 경우 실패로 기록됩니다.
  - **특징 검출 불가**: 입력 데이터가 모두 0이거나 필터의 활성 영역과 전혀 겹치지 않아 두 필터 모두 점수가 0인 경우입니다.
  - **특징 유사도 동일**: 점수가 0은 아니나, 패턴이 대칭적이어서 양쪽 필터에 동일한 가중치가 합산된 경우입니다. (예: 정중앙 점 하나만 찍힌 패턴)
- **규격 불일치 (Dimension Mismatch)**: JSON 헤더의 `size_N` 정보와 실제 데이터 시트의 행렬 크기가 다를 경우 연산을 거부하고 실패 처리합니다. 이는 데이터 오염을 방지하는 안전 장치입니다.
- **결과 불일치 (Logic Error)**: 시뮬레이터는 한 쪽 점수가 높다고 판정했으나, 정답 데이터(Expected)와 상충하는 경우입니다. 주로 필터의 변별력 한계로 인해 발생합니다.
- **필터 데이터 누락 (Missing Assets)**: 특정 규격($N$)에 대한 필터 행렬이 정의되지 않았을 때 발생하며, 시스템의 데이터 무결성 결여를 의미합니다.

### 2. 규격별 성능 추이 및 정밀도 분석 (Performance & Precision)

본 시뮬레이터는 순수 파이썬 반복문을 사용하며, 행렬 크기가 커짐에 따라 다음과 같은 성능 및 정밀도 특성을 보입니다:

- **연산량 증가**: $N^2$ 복잡도에 따라 $3 \times 3$(9회) 대비 $25 \times 25$(625회) 규격은 약 **69.4배**의 연산량 증가가 발생합니다.
- **병목 지점**: 대량의 데이터 처리 시 Python의 리스트 순회 오버헤드가 주된 병목이며, 향후 대규모 NPU 시뮬레이션 시에는 벡터 연산 최적화가 필요함을 시사합니다.
- **1e-9 정밀도(EPSILON) 설정 이유**:
  - **부동소수점 오차 제어**: 컴퓨터는 2진수 체계를 사용하므로 `0.1 + 0.2`가 `0.30000000000000004`로 계산되는 등 미세한 오차가 발생합니다. MAC 연산은 수많은 곱셈과 덧셈이 누적되므로, 이러한 오차가 증폭될 수 있습니다.
  - **판정의 안정성**: 두 MAC 결과값을 단순 비교(`==`)하면 위와 같은 무의미한 소수점 끝자리 차이 때문에 '동점'을 '다름'으로 오판할 수 있습니다. `1e-9`는 이러한 산술적 노이즈를 걸러내고 논리적인 동일성을 보장하기 위한 임계값(Threshold)입니다.
  - **기대값과의 정합성**: 데이터셋의 `UNDECIDED` 상태를 물리적인 점수 일치와 일관되게 연결하기 위해, 충분히 작은 값인 $10^{-9}$를 기준으로 사용하여 판정의 신뢰도를 확보하였습니다.

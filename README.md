# Mini NPU Simulator 개발

## 프로젝트 개요

## 실행 환경

- Python 3.12.1

## 프로젝트 구조

## 실행 방법

```text
프로젝트 루트에서 `python3 main.py` 명령어를 실행하여 프로그램을 시작합니다.
```

## 요구 사항 및 기능 목록

### 데이터 구조

- [ ] NxN 2차원 패턴 및 필터 저장
- [ ] 특정 위치의 값 저장 및 읽기
- [ ] 3x3, 5x5, 13x13, 25x25 크기 처리

### 모드 1 입력 처리(3x3)

- [ ] 3x3 필터 한 줄씩(공백 구분) 입력 및 저장
- [ ] 3x3 패턴 한 줄씩(공백 구분) 입력 및 저장
- [ ] 행/열 수 불일치, 숫자 파싱 실패 시, 안내 문구 출력 및 재입력 유도

### 모드 2 JSON 로드 및 스키마 검증(data.json)

- [ ] data.json 파일 로드
- [ ] filters: size_5, size_13, size_25
- [ ] patterns: size\_{N}\_{idx} 형태의 키, input(2차원 배열), expected 값 포함
- [ ] patterns 각 항목 키에서 N값 추출 후, size_N 필터 선택
- [ ] 필터와 패턴 크기 일치 검증
- [ ] 불일치 시, 해당 케이스 FAIL 처리 및 원인 메시지 출력

### 라벨 정규화(표준화)

- [ ] Cross, X 표준 라벨 사용
- [ ] expected 값: '+' -> Cross, 'x' -> X
- [ ] filter 키 값: 'cross' -> Cross, 'x' -> X
- [ ] 콘솔 출력 및 PASS/FAIL 비교 -> 표준 라벨 기준 수행

### MAC 연산

- [ ] 입력 패턴과 필터 위치별 곱셈 후 덧셈 구현
- [ ] NumPy 등 외부 라이브러리 사용 금지
- [ ] 연산 결과(점수, float 가능) 반환

### 점수 비교 정책(부동소수점/동점 처리 기준)

- [ ] 허용 오차(epsilon) 기반 정책 적용

### 판정 및 PASS/FAIL 출력 규칙

- [ ] 두 필터 점수 비교 후, 판정(A, B, UNDECIDED) 출력
- [ ] 각 패턴별 Cross 필터 점수, X 필터 점수, 판정 출력
- [ ] 판정이 expected와 같으면 PASS, 다르면 FAIL 출력

### 성능 분석

- [ ] 크기별 MAC 연산 ms 단위 시간 측정
- [ ] 크기별 MAC 연산 반복 후 평균 시간 출력(최소 10회)
- [ ] 시간 측정은 I/O 제외, 연산 함수 호출 구간 중심 수행 권장
- [ ] 표 구성: 크기(NxN), 평균 시간(ms), 연산 횟수(N^2)

### 결과 리포트

- [ ] 콘솔 출력
  - [ ] 모드 2 수행 시, 전체 테스트 수/통과 수/실패 수 출력
  - [ ] 실패 케이스, 케이스 식별자와 실패 사유 요약 출력
- [ ] README 작성
  - [ ] 결과 리포트 섹션, 실패 원인 분석 + 시간 복잡도 분석 10줄 이상
  - [ ] 실패 0개일 경우, "왜 실패가 0인지(정규화, epsilon 정책)" 등 요약

### 실행 흐름

- [ ] 모드 1
  - [ ] 3x3 필터 A, B 입력
  - [ ] 저장 확인
  - [ ] 3x3 패턴 입력
  - [ ] MAC 연산
  - [ ] 결과 판정
  - [ ] 성능 분석
- [ ] 모드 2
  - [ ] 필터 로드
  - [ ] 패턴 로드 및 검증
  - [ ] MAC 연산
  - [ ] 결과 판정
  - [ ] PASS/FAIL 출력
  - [ ] 성능 분석
  - [ ] 결과 요약 출력

### 시뮬레이터 최적화

- [ ] 2차원 배열 -> 1차원 배열(N^2) 변환 후 접근
- [ ] 최적화 전/후 성능 분석(동일 입력, 동일 반복 횟수)

### 패턴 생성기 개발

- [ ] 크기 N 입력 시, N x N 크기의 Cross와 X 패턴 생성
- [ ] 생성된 패턴을 모드 1 성능 분석에 재활용

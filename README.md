# Mini NPU Simulator 개발

## 프로젝트 개요

간단한 NPU의 핵심 연산인 MAC 연산을 시뮬레이션하고, 입력된 패턴이 어떤 필터와 더 유사한지 판정하며 성능을 분석하는 프로그램입니다.

## 실행 환경

- Python 3.12.1

## 프로젝트 구조

```text
.
├── main.py          # 프로그램 진입점 및 모드 선택
├── controllers.py   # 전체 실행 흐름 제어 (모드 1, 모드 2)
├── models.py        # MAC 연산 및 판별 핵심 로직 (NPUSimulator)
├── views.py         # 터미널 UI 출력 및 데이터 시각화 (TerminalView)
├── utils.py         # JSON 로드, 라벨 정규화 등 유틸리티
├── constants.py     # EPSILON, 반복 횟수 등 전역 상수 관리
├── data.json        # 모드 2 전용 분석 데이터셋 (15개 케이스)
├── input.txt        # 모드 1 테스트를 위한 자동 입력 더미 파일
└── README.md        # 프로젝트 문서
```

## 실행 방법

```text
프로젝트 루트에서 `python3 main.py` 명령어를 실행하여 프로그램을 시작합니다.
```

## 요구 사항 및 기능 목록

### 데이터 구조

- [x] NxN 2차원 패턴 및 필터 저장
- [x] 특정 위치의 값 저장 및 읽기
- [x] 3x3, 5x5, 13x13, 25x25 크기 처리

### 모드 1 입력 처리(3x3)

- [x] 3x3 필터 한 줄씩(공백 구분) 입력 및 저장
- [x] 3x3 패턴 한 줄씩(공백 구분) 입력 및 저장
- [x] 행/열 수 불일치, 숫자 파싱 실패 시 안내 문구 출력 및 재입력 유도

### 모드 2 JSON 로드 및 스키마 검증(data.json)

- [x] data.json 파일 로드 및 전처리 (수치 데이터 float 일괄 변환)
- [x] filters: size_3, size_5, size_13, size_25 지원
- [x] patterns: size*{N}*{idx} 키 구조 지원 (정교한 2단계 숫자 정렬 적용)
- [x] 필터와 패턴 크기 일치 검증 및 예외 처리

### 라벨 정규화(표준화)

- [x] Cross, X 표준 라벨 사용
- [x] '+', 'cross' -> Cross / 'x' -> X 정규화
- [x] 'undecided' -> UNDECIDED 표준 상태 정규화

### MAC 연산

- [x] 입력 패턴과 필터 위치별 곱셈 후 덧셈 (O(N^2))
- [x] NumPy 등 외부 라이브러리 사용 금지 및 순수 파이썬 구현

### 판정 및 출력 규칙

- [x] 허용 오차(epsilon = 1e-9) 기반 동점 판정 정책
- [x] 모드 1 판정: `A`, `B`, `UNDECIDED`
- [x] 모드 2 출력 구조: `판정(X/Cross/-)`, `기대값`, `결과(PASS/FAIL)`
- [x] 기대값이 'UNDECIDED'인 경우, 동점 발생 시 PASS 처리

### 성능 분석 및 결과 리포트

- [x] 크기별 MAC 연산 100회 반복 평균 시간 측정 (ms 단위)
- [x] 규격별 성능 통계 표(Table) 출력
- [x] 모드 2 전체 테스트 통계 및 실패 케이스 사유 요약 브리핑

## 결과 리포트 및 분석

### 1. 실패 원인 분석

모드 2 분석 중 발생하는 실패 케이스는 다음과 같이 분류됩니다:

- **동점 발생 (FAIL)**: Cross 필터와 X 필터의 MAC 점수 차이가 `EPSILON(1e-9)` 미만이면서 기대값이 `UNDECIDED`가 아니라면 실패로 처리합니다.
- **규격 불일치 (FAIL)**: JSON에 정의된 N 규격과 실제 입력된 행렬의 크기가 일치하지 않는 경우, 즉시 실패 처리합니다.
- **결과 불일치 (FAIL)**: 시뮬레이터의 판정(Cross/X)이 데이터셋의 정답(Expected)과 다른 경우, 실패로 처리합니다.
- **필터 데이터 없음 (FAIL)**: 패턴 규격에 맞는 필터 데이터가 `data.json` 내에 존재하지 않는 경우, 실패로 처리합니다.

### 2. 시간 복잡도 분석

- **시간 복잡도**: $N \times N$ 크기의 행렬을 순회하며 원소별 연산을 수행하므로 **$O(N^2)$**입니다. 데이터 크기($N$)가 커질수록 연산량은 제곱에 비례하여 증가하며, 이는 성능 분석 표의 25x25 연산 시간 증가로 알 수 있습니다.

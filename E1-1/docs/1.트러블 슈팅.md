### 1. Docker Volume 매핑 및 데이터 휘발 문제

- **문제**
  - 컨테이너 내부의 `/test_volume` 디렉토리에 파일을 생성했으나, 컨테이너를 삭제 후 재실행하면 이전 데이터가 유지되지 않고 사라짐.
  - **발생 시 로그:**

    ```zsh
    # 첫 번째 컨테이너 실행 및 파일 생성
    $ docker run -it -v test_volume --name volume_test ubuntu
    root@c74efa587eee:/# echo "Is this file alive?" > /test_volume/test.txt
    root@c74efa587eee:/# exit

    # 컨테이너 삭제 후 새로운 시도
    $ docker run -it -v test_volume --name volume_check ubuntu
    root@95589c58e0a3:/# ls /test_volume/
    # (결과 없음: 데이터 사라짐)
    ```

- **원인 가설**
  - `-v test_volume` 옵션 사용 시, 콜론(`:`) 구분이 없어 Docker 내의 폴더에 매핑이 되지 않았음.
  - 이로 인해 컨테이너 재실행 시마다 새로운 익명 볼륨이 생성되어 데이터가 연결되지 않음.

- **확인**
  - 명령어 해석: `-v test_volume`은 Docker에 의해 `"이름 없는 볼륨을 /test_volume에 마운트"`하라는 명령으로 해석됨.
  - 결과적으로 이전 데이터와 연결되지 않으며, 매번 새로운 독립 볼륨이 할당됨을 확인.

- **해결/대안**
  - 볼륨 이름과 컨테이너 내부 경로를 콜론(`:`)으로 명확히 매핑함.

  ```zsh
  # 올바른 문법: -v [볼륨명]:[컨테이너 내부 경로]
  $ docker run -it -v test_volume:/test_volume --name volume_test ubuntu
  ```

### 2. 포트 충돌로 인한 컨테이너 실행 실패 문제

- **문제**
  - Nginx 컨테이너를 8000번 포트로 실행하려 했으나, 포트 점유 에러와 함께 실행이 거부됨.
  - **발생 시 로그:**
    ```zsh
    $ docker run -d -p 8000:80 nginx:alpine
    docker: Error response from daemon: failed to setup container networking: driver failed programming external connectivity on endpoint ... : Bind for 0.0.0.0:8000 failed: port is already allocated
    ```

- **원인 가설**
  - 로컬 호스트의 8000번 포트를 이미 다른 프로세스나 이전에 실행 중인 Docker 컨테이너가 점유하고 있음.

- **확인**
  - `docker ps` 명령어로 현재 8000번 포트를 사용 중인 기존 컨테이너 확인:
    ```zsh
    $ docker ps
    CONTAINER ID   IMAGE          ...   PORTS                                   NAMES
    982c97d82a42   nginx:alpine   ...   0.0.0.0:8000->80/tcp, [::]:8000->80/tcp   condescending_proskuriakova
    ```

- **해결/대안**
  - **선택 방법: 기존 컨테이너 제거 후 재실행**
    ```zsh
    $ docker kill 982c97d82a42
    $ docker run -d -p 8000:80 nginx:alpine
    ```
  - **대안: 다른 호스트 포트 할당**
    ```zsh
    $ docker run -d -p 8001:80 nginx:alpine
    ```
